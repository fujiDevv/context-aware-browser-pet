# Security Policy

## Supported Versions

Currently, only the latest release of the extension is supported with active security updates.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Security is a top priority for this project. If you discover a security vulnerability, please **DO NOT** open a public issue.

Instead, please report it via one of the following methods:
1. **GitHub Security Advisories:** Navigate to the "Security" tab in this repository and click "Report a vulnerability" to open a private advisory.
2. **Direct Contact:** If you are unable to use the GitHub UI, please contact the repository maintainer directly.

### What to Include
When reporting an issue, please provide as much context as possible, including:
- A clear description of the vulnerability and its potential impact.
- Detailed steps to reproduce the issue.
- Details about your environment (e.g., Chrome version, OS).
- Any proposed mitigation if you have one.

### Response Timeline
We will acknowledge receipt of your vulnerability report within 48 hours and strive to provide a timeline for the fix. Once the vulnerability is resolved, you will be credited in the release notes (unless you prefer to remain anonymous).
